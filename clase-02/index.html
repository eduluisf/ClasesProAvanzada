Módulo 01

¿Qué es una función?

Un bloque de código con nombre que podemos reutilizar

Funciones

Estructura básica

Partes de una función

  • def → palabra clave que define la función
  • nombre → identificador con el que la llamamos
  • parámetros → datos que recibe (pueden ser ninguno)
  • cuerpo → instrucciones indentadas
  • return → valor que devuelve (opcional)

Definir ≠ Ejecutar

  • Escribir def solo prepara la función
  • Para ejecutarla hay que llamarla: nombre()
  • Se puede llamar tantas veces como se quiera
# Sin parámetros
def saludar():
    print("Hola mundo")

saludar()   # Hola mundo
saludar()   # Hola mundo  (llamada 2)

# Con un parámetro
def saludar_a(nombre):
    print(f"Hola, {nombre}!")

saludar_a("Ana")    # Hola, Ana!
saludar_a("Luis")   # Hola, Luis!

# Con varios parámetros y return
def sumar(a, b):
    return a + b

resultado = sumar(3, 5)
print(resultado)      # 8
Funciones

¿Por qué usar funciones?

Sin funciones — código repetido

# Mismo cálculo copiado 3 veces
nota1 = (8 + 9 + 7) / 3
print(f"Promedio: {nota1:.1f}")

nota2 = (6 + 10 + 8) / 3
print(f"Promedio: {nota2:.1f}")

nota3 = (5 + 7 + 9) / 3
print(f"Promedio: {nota3:.1f}")
# Si cambia el cálculo → editar 3 lugares

Con funciones — DRY (Don't Repeat Yourself)

def promedio(n1, n2, n3):
    resultado = (n1 + n2 + n3) / 3
    print(f"Promedio: {resultado:.1f}")

promedio(8, 9, 7)   # Promedio: 8.0
promedio(6, 10, 8)  # Promedio: 8.0
promedio(5, 7, 9)   # Promedio: 7.0
# Cambio → editar 1 sola vez

Principio DRY

  • Reutilización — escribir una vez, usar muchas
  • Legibilidad — el nombre dice qué hace
  • Mantenimiento — un cambio aplica a todo
Módulo 02

Parámetros y Argumentos

Cómo enviar datos a una función

Parámetros

Posicionales vs Keyword

Parámetro vs Argumento

  • Parámetro: variable en la definición de la función
  • Argumento: valor real que se pasa al llamarla

Posicionales — el orden importa

  • Se asignan de izquierda a derecha
  • Deben pasarse todos

Keyword — el nombre importa

  • Se nombra explícitamente: param=valor
  • El orden no importa
  • Más legible en funciones con muchos parámetros
def presentar(nombre, edad, ciudad):
    print(f"{nombre}, {edad}, {ciudad}")

# Posicional — orden importa
presentar("Ana", 20, "Lima")
# Ana, 20, Lima

# Keyword — orden libre
presentar(ciudad="Lima", nombre="Ana", edad=20)
# Ana, 20, Lima

# Mixto — posicional primero, keyword al final
presentar("Ana", ciudad="Lima", edad=20)
# Ana, 20, Lima
Parámetros

Valores por defecto

¿Qué son?

  • Valor que toma el parámetro si no se pasa al llamar
  • Los parámetros con defecto van siempre al final
  • Permiten hacer argumentos opcionales

Error clásico: lista como defecto

  • Las listas son mutables → se comparten entre llamadas
  • Usa None y crea la lista dentro
def saludar(nombre, saludo="Hola", puntos="!"):
    print(f"{saludo}, {nombre}{puntos}")

saludar("Ana")                # Hola, Ana!
saludar("Luis", "Buenos días") # Buenos días, Luis!
saludar("Mia", puntos=".")   # Hola, Mia.

# BIEN — None como centinela
def agregar(x, lista=None):
    if lista is None:
        lista = []
    lista.append(x)
    return lista
Parámetros

Type hints — anotar los tipos

¿Qué son?

  • Anotaciones que indican qué tipo espera cada parámetro
  • Y qué tipo va a retornar la función (-> tipo)
  • Python no las obliga — son una guía para el programador
  • Mejoran mucho la legibilidad del código

Sintaxis

def nombre(param: tipo) -> tipo_retorno:

Tipos más comunes

  • int — número entero
  • float — número decimal
  • str — texto
  • bool — True / False
  • list — lista
  • None — cuando no retorna nada
# Sin type hints — ¿qué tipo recibe/retorna?
def calcular_imc(peso, altura):
    return peso / altura ** 2

# Con type hints — queda claro al leer
def calcular_imc(peso: float, altura: float) -> float:
    return peso / altura ** 2

def es_par(n: int) -> bool:
    return n % 2 == 0

def saludar(nombre: str, veces: int = 1) -> None:
    for _ in range(veces):
        print(f"Hola, {nombre}!")

def dividir(a: float, b: float) -> float:
    return a / b

# Python no fuerza el tipo — esto funciona igual
print(es_par(4))    # True
print(es_par(7))    # False
saludar("Ana", 3)  # Hola, Ana! x3
Módulo 03

Funciones con return

La diferencia entre imprimir y devolver un valor

Return

Sin return vs con return

Sin return — solo imprime

  • Realiza una acción (imprime, modifica)
  • Retorna None de forma implícita
  • El valor no se puede guardar ni usar
def mostrar_doble(x):
    print(x * 2)

mostrar_doble(5)       # imprime 10
res = mostrar_doble(5)
print(res)              # None !
print(res + 1)         # Error — None + 1

Con return — devuelve valor

  • El valor se puede guardar, operar, pasar
  • El código tras return no se ejecuta
  • Puede retornar cualquier tipo
def calcular_doble(x):
    return x * 2

res = calcular_doble(5)
print(res)                       # 10
print(res + 1)                  # 11
print(calcular_doble(3) * 4)   # 24
Return

Múltiples valores y return anticipado

Retornar varios valores

  • Python permite retornar varios valores separados por coma
  • Internamente se convierte en una tupla
  • Se puede desempaquetar con múltiples variables
def min_max(lista):
    return min(lista), max(lista)

menor, mayor = min_max([3, 1, 9, 2])
print(menor, mayor)  # 1  9

Return anticipado — guardia

  • Salir temprano si se cumple una condición
  • Evita if/else anidados
  • Código más plano y legible
def dividir(a, b):
    if b == 0:
        return None   # retorno anticipado
    return a / b

print(dividir(10, 2))   # 5.0
print(dividir(10, 0))   # None
Módulo 04

Recursión

Una función que se llama a sí misma para resolver un problema dividiéndolo en partes más pequeñas

Recursión

¿Qué es la recursión?

Idea central

Una función resuelve un problema grande resolviéndolo en términos del mismo problema más pequeño, hasta llegar a un caso tan simple que la respuesta es directa.

Analogía

Para saber cuántas personas hay en una fila detrás de ti: preguntas a la persona de atrás, ella pregunta a la siguiente... hasta que el último dice "soy 1". Cada uno suma 1 a la respuesta del siguiente.

Dos partes obligatorias

  • Caso base: condición de parada. El problema es tan pequeño que la respuesta es directa, sin otra llamada recursiva.
  • Caso recursivo: la función se llama a sí misma con un input más pequeño.

Sin caso base = loop infinito

Python lanza RecursionError al llegar al límite (~1000 llamadas).

Recursión

Factorial — paso a paso

def factorial(n):
    if n == 0 or n == 1:  # caso base
        return 1
    return n * factorial(n - 1)  # recursivo

print(factorial(5))  # 120

¿Por qué funciona?

  • factorial(5) necesita factorial(4)
  • Cada llamada espera a que la siguiente termine
  • Cuando llega a factorial(1) → retorna 1
  • Las respuestas se propagan hacia atrás

Traza de ejecución

factorial(1) → 1
factorial(2) → 2 × 1 = 2
factorial(3) → 3 × 2 = 6
factorial(4) → 4 × 6 = 24
factorial(5) → 5 × 24 = 120

Las llamadas se apilan, luego se resuelven de abajo hacia arriba.

Recursión

La pila de llamadas (call stack)

¿Qué es el call stack?

  • Cada llamada a función crea un frame en memoria
  • Los frames se apilan uno sobre otro
  • Cuando la función retorna, su frame se elimina
  • La recursión apila muchos frames → puede agotarse

RecursionError

  • Pasa cuando olvidamos el caso base
  • O cuando el problema no se hace más pequeño
  • Límite por defecto: ~1000 frames
# factorial(4) — visualización de la pila

# FASE 1: construcción (bajada)
# factorial(4)  ← espera factorial(3)
#   factorial(3)  ← espera factorial(2)
#     factorial(2)  ← espera factorial(1)
#       factorial(1)  → retorna 1  ✓ caso base

# FASE 2: resolución (subida)
#     factorial(2)  → 2 × 1 = 2
#   factorial(3)  → 3 × 2 = 6
# factorial(4)  → 4 × 6 = 24

# Sin caso base — pila infinita
def infinita(n):
    return n * infinita(n - 1)  # ← nunca para
# RecursionError: maximum recursion depth exceeded
Recursión

Fibonacci

Definición matemática

  • fib(0) = 0 → caso base
  • fib(1) = 1 → caso base
  • fib(n) = fib(n-1) + fib(n-2) → recursivo

Problema de eficiencia

fib(5) llama a fib(3) y fib(4). fib(4) llama a fib(3) otra vez. Se recalcula el mismo valor muchas veces.

# Fibonacci recursivo — elegante pero lento
def fib(n):
    if n <= 1:          # caso base
        return n
    return fib(n-1) + fib(n-2)

for i in range(8):
    print(fib(i), end=" ")
# 0 1 1 2 3 5 8 13

# Fibonacci iterativo — eficiente
def fib_iter(n):
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a
Recursión

Recursión con listas

Patrón común

  • Caso base: lista vacía []
  • Caso recursivo: procesar lista[0] + llamarse con lista[1:]
  • lista[1:] es la lista sin el primer elemento
# Suma recursiva
def suma(lista):
    if lista == []:    # caso base
        return 0
    return lista[0] + suma(lista[1:])

print(suma([1, 2, 3, 4]))  # 10
# Traza: suma([1, 2, 3, 4])
# → 1 + suma([2, 3, 4])
# → 1 + 2 + suma([3, 4])
# → 1 + 2 + 3 + suma([4])
# → 1 + 2 + 3 + 4 + suma([])
# → 1 + 2 + 3 + 4 + 0 = 10

# Buscar un elemento
def contiene(lista, x):
    if lista == []:
        return False
    if lista[0] == x:
        return True
    return contiene(lista[1:], x)

print(contiene([1, 2, 3], 2))  # True
print(contiene([1, 2, 3], 9))  # False
Recursión

Errores comunes

1. Sin caso base

# MAL — nunca para
def factorial_mal(n):
    return n * factorial_mal(n - 1)
# RecursionError

# BIEN
def factorial(n):
    if n <= 1: return 1   # ← caso base
    return n * factorial(n - 1)

2. El problema no se hace más pequeño

# MAL — n nunca cambia
def contar_mal(n):
    print(n)
    return contar_mal(n)  # ← mismo n

3. Caso base incorrecto

# MAL — no cubre n=0
def factorial_mal2(n):
    if n == 1: return 1
    return n * factorial_mal2(n - 1)

factorial_mal2(0)  # RecursionError!

# BIEN — cubre 0 y 1
def factorial(n):
    if n <= 1: return 1
    return n * factorial(n - 1)

Checklist antes de escribir recursión

  1. ¿Definí el caso base?
  2. ¿El input se hace más pequeño cada vez?
  3. ¿El caso base cubre todos los casos límite?

Ejercicio 1 — Return con decisión
Solución

Clasificar una nota

def clasificar_nota(nota: float) -> str:
    if nota >= 9:
        return "Excelente"
    if nota >= 7:
        return "Aprobado"
    if nota >= 5:
        return "Recuperación"
    return "Reprobado"

# Guardar el return y usarlo
alumnos = [("Ana", 9.5), ("Luis", 6.0), ("Mia", 3.8)]

for nombre, nota in alumnos:
    categoria = clasificar_nota(nota)   # ← guardamos el return
    print(f"{nombre}: {categoria}")

# Ana: Excelente
# Luis: Recuperación
# Mia: Reprobado

Puntos clave

  • Cada return anticipado sale inmediatamente de la función
  • No se necesita elif — si llegó al segundo if es porque nota < 9
  • El valor retornado se guarda en categoria y se usa en el print

Solución

Calcular e interpretar el IMC

def calcular_imc(peso: float, altura: float) -> float:
    return peso / altura ** 2

def clasificar_imc(imc: float) -> str:
    if imc < 18.5: return "Bajo peso"
    if imc < 25:   return "Normal"
    if imc < 30:   return "Sobrepeso"
    return "Obesidad"

# Encadenar: return de 1 → argumento de 2
imc = calcular_imc(70, 1.75)
categoria = clasificar_imc(imc)
print(f"IMC: {imc:.1f} → {categoria}")
# IMC: 22.9 → Normal

# O en una sola línea
print(clasificar_imc(calcular_imc(90, 1.70)))
# Sobrepeso

  • El float que retorna calcular_imc se guarda en imc
  • Ese valor se pasa como argumento a clasificar_imc
  • También se puede pasar directo sin guardar
  • Los type hints documentan exactamente qué entra y qué sale

Ejercicio 3 — Return en expresiones
Solución

Conversor de temperaturas

def celsius_a_fahrenheit(c: float) -> float:
    return c * 9 / 5 + 32

def fahrenheit_a_celsius(f: float) -> float:
    return (f - 32) * 5 / 9

# Usar el return directamente
print(celsius_a_fahrenheit(0))    # 32.0  (punto de congelación)
print(celsius_a_fahrenheit(100))  # 212.0 (punto de ebullición)
print(fahrenheit_a_celsius(98.6)) # 37.0  (temperatura corporal)

# Guardar results en una lista
temps_c = [0, 20, 37, 100]
temps_f = [celsius_a_fahrenheit(t) for t in temps_c]
print(temps_f)
# [32.0, 68.0, 98.6, 212.0]

Puntos clave

  • El return se puede usar dentro de print directamente
  • O guardarse en una variable para reusarlo
  • O usarse en una list comprehension
  • Type hints float -> float dejan claro que entra y sale un decimal
Ejercicio 4 — Recursión
Enunciado

Potencia recursiva

Objetivo

Crea una función recursiva potencia(base: int, exp: int) -> int que calcule baseexp sin usar el operador **.

Pistas

  • Caso base: exp == 0 → retorna 1 (cualquier número a la 0 = 1)
  • Caso recursivo: base × potencia(base, exp - 1)
  • potencia(2, 3) = 2 × 2 × 2 = 8

Intenta resolverlo
antes de pasar al siguiente slide

Ejercicio 4 — Recursión
Solución

Potencia recursiva

def potencia(base: int, exp: int) -> int:
    if exp == 0:
        return 1
    return base * potencia(base, exp - 1)

print(potencia(2, 3))   # 8
print(potencia(5, 0))   # 1
print(potencia(3, 4))   # 81
potencia(2, 0) → 1
potencia(2, 1) → 2 × 1 = 2
potencia(2, 2) → 2 × 2 = 4
potencia(2, 3) → 2 × 4 = 8

Puntos clave

  • exp se reduce en 1 cada llamada → llega al caso base
  • El return de cada nivel se multiplica por base
  • Nótese que el return también puede usarse en cálculos más grandes
# Usar el return en una expresión
area = potencia(5, 2)         # 25
volumen = potencia(3, 3)      # 27
print(area + volumen)          # 52
Módulo 05

Argumentos múltiples

*args y **kwargs — cantidad variable de argumentos

Argumentos múltiples

*args — cantidad variable de valores

¿Qué hace?

  • Recibe cualquier cantidad de argumentos posicionales
  • Los empaca en una tupla
  • El * es lo importante, no el nombre args

¿Cuándo usarlo?

  • Funciones que aceptan n elementos sin saber cuántos
  • Ej: suma, promedio, imprimir varios items
def sumar(*numeros):
    total = 0
    for n in numeros:
        total += n
    return total

print(sumar(1, 2))            # 3
print(sumar(1, 2, 3, 4))      # 10
print(sumar(1, 2, 3, 4, 5))   # 15

# Combinar con parámetro normal
def presentar(titulo, *nombres):
    print(f"--- {titulo} ---")
    for n in nombres:
        print(f"  {n}")

presentar("Alumnos", "Ana", "Luis", "Mia")
Argumentos múltiples

**kwargs — clave=valor variable

¿Qué hace?

  • Recibe argumentos nombrados extras
  • Los empaca en un diccionario
  • El ** es lo importante, no el nombre kwargs

Orden en la firma

def f(normal, *args, **kwargs)

  • Primero los parámetros normales
  • Luego *args
  • Último **kwargs
def perfil(nombre, **datos):
    print(f"Perfil de {nombre}:")
    for clave, valor in datos.items():
        print(f"  {clave}: {valor}")

perfil("Ana", edad=20, ciudad="Lima", nota=9)
# Perfil de Ana:
#   edad: 20
#   ciudad: Lima
#   nota: 9

perfil("Luis", edad=22)
# Perfil de Luis:
#   edad: 22
Resumen

Lo que aprendimos

Funciones

  • def nombre(params)
  • Definir ≠ ejecutar
  • Sin / con return
  • Parámetros y valores por defecto

Recursión

  • Caso base + caso recursivo
  • El input se hace más pequeño
  • Call stack — frames apilados
  • Factorial, Fibonacci, listas

Argumentos múltiples

  • *args → tupla de valores
  • **kwargs → dict de clave=valor
  • Orden: normal → *args → **kwargs

Buenas prácticas

  • Usar return para reutilizar valores
  • Siempre definir caso base en recursión
  • DRY: no repetir lógica
