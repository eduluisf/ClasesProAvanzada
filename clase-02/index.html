Módulo 01

Parámetros y Argumentos

Cómo pasar datos a una función

Parámetros

Posicionales vs Keyword

Posicionales

  • El orden importa
  • Se asignan por posición al llamar
  • Deben pasarse todos

Keyword (por nombre)

  • El orden no importa
  • Se nombra el parámetro al llamar
  • Más legible en funciones largas
def crear_usuario(nombre, edad, rol):
    print(f"{nombre}, {edad}, {rol}")

# Posicional — orden importa
crear_usuario("Ana", 25, "admin")

# Keyword — orden libre
crear_usuario(rol="user", nombre="Luis", edad=30)

# Mixto — posicional primero
crear_usuario("Mia", rol="admin", edad=22)
Parámetros

Valores por defecto

Regla de oro

  • Los parámetros con defecto van al final
  • Si no se pasa el argumento, usa el valor por defecto
  • Permite hacer funciones opcionales

Error común

  • Nunca uses una lista o dict como defecto
  • Usa None y crea el objeto dentro
def saludar(nombre, saludo="Hola"):
    print(f"{saludo}, {nombre}!")

saludar("Ana")              # Hola, Ana!
saludar("Luis", "Buenas") # Buenas, Luis!

# MAL — lista mutable como defecto
def agregar_mal(x, lista=[]):
    lista.append(x)
    return lista

# BIEN — None como centinela
def agregar_bien(x, lista=None):
    if lista is None:
        lista = []
    lista.append(x)
    return lista
Parámetros

*args y **kwargs

*args — cantidad variable de valores

  • Recibe cualquier cantidad de argumentos
  • Se convierte en una tupla
  • El * es lo importante, no el nombre
def sumar(*nums):
    return sum(nums)

print(sumar(1, 2))          # 3
print(sumar(1, 2, 3, 4))   # 10

**kwargs — cantidad variable de clave=valor

  • Recibe argumentos nombrados extras
  • Se convierte en un diccionario
def perfil(nombre, **datos):
    print(f"Nombre: {nombre}")
    for k, v in datos.items():
        print(f"  {k}: {v}")

perfil("Ana", edad=25, ciudad="Lima")
# Nombre: Ana
#   edad: 25
#   ciudad: Lima
Módulo 02

Funciones Lambda

Funciones anónimas en una sola línea

Lambda

¿Qué es una lambda?

Sintaxis

lambda params : expresión

Características

  • Una sola expresión (no sentencias)
  • Retorna el valor automáticamente
  • Ideal para funciones simples y cortas
  • Se usa donde se necesita una función momentánea
# Función normal vs lambda
def doble(x):
    return x * 2

doble_l = lambda x: x * 2

print(doble(5))    # 10
print(doble_l(5)) # 10

# Lambda con varios parámetros
suma = lambda a, b: a + b
print(suma(3, 4))  # 7

# Lambda con condición
par = lambda n: "par" if n % 2 == 0 else "impar"
print(par(7))  # impar
Lambda

map() y filter()

map(funcion, lista)

  • Aplica la función a cada elemento
  • Retorna un nuevo iterable con los resultados
  • No modifica la lista original
nums = [1, 2, 3, 4, 5]

cuadrados = list(map(lambda x: x**2, nums))
print(cuadrados)  # [1, 4, 9, 16, 25]

mayus = list(map(str.upper, ["ana", "luis"]))
print(mayus)  # ['ANA', 'LUIS']

filter(funcion, lista)

  • Conserva solo los elementos donde la función retorna True
  • Filtra la lista según condición
nums = [1, 2, 3, 4, 5, 6]

pares = list(filter(lambda x: x % 2 == 0, nums))
print(pares)  # [2, 4, 6]

mayores = list(filter(lambda x: x > 3, nums))
print(mayores)  # [4, 5, 6]

# Equivalente con list comprehension
pares_lc = [x for x in nums if x % 2 == 0]
Módulo 03

Scope (Alcance)

Dónde vive cada variable

Scope

Local vs Global

Regla LEGB

  • Local → dentro de la función actual
  • Enclosing → función que la contiene
  • Global → nivel del módulo
  • Built-in → funciones de Python

Buena práctica

  • Evita usar global — pasa datos por parámetros
  • Las funciones deben depender de sus entradas, no de variables externas
x = 10  # variable global

def ver_x():
    print(x)   # puede LEER la global

def cambiar_x():
    global x   # declara que usará la global
    x = 99

def local_x():
    x = 50     # crea una variable LOCAL, no afecta la global
    print(x)   # 50

ver_x()      # 10
local_x()    # 50
ver_x()      # 10  ← global no cambió
cambiar_x()
ver_x()      # 99  ← ahora sí cambió
Scope

Funciones anidadas y nonlocal

Función anidada (closure)

  • Una función definida dentro de otra
  • Puede acceder a variables de la función exterior
  • nonlocal permite modificar esa variable

¿Para qué sirve?

  • Crear funciones fábrica (factory)
  • Encapsular estado sin usar clases
  • Decoradores (tema avanzado)
# Closure básico
def contador():
    cuenta = 0
    def incrementar():
        nonlocal cuenta
        cuenta += 1
        print(cuenta)
    return incrementar

c = contador()
c()  # 1
c()  # 2
c()  # 3

# Función fábrica
def multiplicador(factor):
    return lambda x: x * factor

doble  = multiplicador(2)
triple = multiplicador(3)
print(doble(5))   # 10
print(triple(5))  # 15
Módulo 04

Recursión

Una función que se llama a sí misma

Recursión

Anatomía de una función recursiva

Dos partes obligatorias

  • Caso base → condición de parada (sin ella, loop infinito)
  • Caso recursivo → se llama a sí misma con un problema más pequeño

Limitación en Python

  • Límite de ~1000 llamadas recursivas por defecto
  • Para problemas grandes → usar iteración
# Factorial: n! = n × (n-1) × … × 1
def factorial(n):
    if n == 0 or n == 1:   # caso base
        return 1
    return n * factorial(n - 1)  # recursivo

print(factorial(5))  # 120
# factorial(5)
# → 5 × factorial(4)
# → 5 × 4 × factorial(3)
# → 5 × 4 × 3 × factorial(2)
# → 5 × 4 × 3 × 2 × 1  = 120
Recursión

Recursión vs Iteración

Fibonacci Recursivo
def fib_r(n):
    if n <= 1:
        return n
    return fib_r(n-1) + fib_r(n-2)

print(fib_r(10))  # 55
# Elegante pero lento para n grande

Recursivo

  • Código mas legible
  • Más lento (re-calcula)
  • Riesgo de stack overflow
Fibonacci Iterativo
def fib_i(n):
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a

print(fib_i(10))  # 55
# Rápido y sin límite de pila

Iterativo

  • Más eficiente
  • Sin límite de profundidad
  • Ideal para problemas grandes
Módulo 05

Funciones de Orden Superior

Funciones que reciben o retornan otras funciones

Orden Superior

Funciones como argumento

sorted() con key=

  • sorted() acepta una función como criterio de ordenamiento
  • Se usa key=lambda ... para personalizar
nombres = ["Zara", "Ana", "Miguel", "Lu"]

# Ordenar por longitud
print(sorted(nombres, key=lambda x: len(x)))
# ['Lu', 'Ana', 'Zara', 'Miguel']

# Ordenar por última letra
print(sorted(nombres, key=lambda x: x[-1]))
# ['Zara', 'Ana', 'Miguel', 'Lu']
# Lista de diccionarios
alumnos = [
    {"nombre": "Ana",   "nota": 9},
    {"nombre": "Luis",  "nota": 7},
    {"nombre": "Maria", "nota": 10},
]

# Ordenar por nota de mayor a menor
top = sorted(alumnos, key=lambda a: a["nota"], reverse=True)

for a in top:
    print(a["nombre"], a["nota"])
# Maria 10
# Ana   9
# Luis  7
Orden Superior

Funciones que retornan funciones

Patrón fábrica

  • Una función crea y retorna otra función
  • La función retornada recuerda el contexto de creación
  • Muy usado en validadores y configuradores
def crear_saludo(idioma):
    if idioma == "es":
        return lambda n: f"Hola, {n}!"
    elif idioma == "en":
        return lambda n: f"Hello, {n}!"
    else:
        return lambda n: f"Hi, {n}!"

saludar_es = crear_saludo("es")
saludar_en = crear_saludo("en")

print(saludar_es("Ana"))   # Hola, Ana!
print(saludar_en("John"))  # Hello, John!

Ejercicio 1 — *args
Enunciado

Calculadora flexible

Objetivo

Crea una función estadisticas(*nums) que reciba cualquier cantidad de números y retorne un diccionario con: suma, promedio, mínimo y máximo.

Ejemplo esperado

estadisticas(3, 7, 2, 9, 1)
→ {"suma": 22, "promedio": 4.4,
   "min": 1, "max": 9}

Intenta resolverlo
antes de pasar al siguiente slide

Ejercicio 1 — *args
Solución

Calculadora flexible

def estadisticas(*nums):
    return {
        "suma":     sum(nums),
        "promedio": sum(nums) / len(nums),
        "min":      min(nums),
        "max":      max(nums),
    }

resultado = estadisticas(3, 7, 2, 9, 1)
print(resultado)
# {'suma': 22, 'promedio': 4.4, 'min': 1, 'max': 9}

Puntos clave

  • *nums convierte todos los args en tupla
  • sum(), min(), max() aceptan cualquier iterable
  • La función sirve para cualquier cantidad de números
Ejercicio 2 — Lambda + sorted
Enunciado

Ranking de estudiantes

Objetivo

Dada la lista de alumnos, usa sorted() con una lambda para ordenarlos de mayor a menor nota. Luego imprime el podio (top 3) con su posición.

alumnos = [
    {"nombre": "Carlos", "nota": 8},
    {"nombre": "Sofia",  "nota": 10},
    {"nombre": "Pedro",  "nota": 7},
    {"nombre": "Lucia",  "nota": 9},
    {"nombre": "Tomas",  "nota": 6},
]

Intenta resolverlo
antes de pasar al siguiente slide

Ejercicio 2 — Lambda + sorted
Solución

Ranking de estudiantes

ranking = sorted(
    alumnos,
    key=lambda a: a["nota"],
    reverse=True
)

for i, a in enumerate(ranking[:3], 1):
    print(f"#{i} {a['nombre']} — {a['nota']}")

# #1 Sofia  — 10
# #2 Lucia  — 9
# #3 Carlos — 8

Puntos clave

  • key=lambda a: a["nota"] extrae la nota para comparar
  • reverse=True invierte el orden (mayor primero)
  • enumerate(..., 1) empieza a contar desde 1
  • ranking[:3] toma solo los primeros 3
Ejercicio 3 — Recursión
Enunciado

Suma recursiva de una lista

Objetivo

Crea una función recursiva suma_lista(lista) que calcule la suma de todos los elementos de una lista sin usar sum() ni ciclos.

Pista

  • Caso base: lista vacía → retorna 0
  • Caso recursivo: lista[0] + suma_lista(lista[1:])

Intenta resolverlo
antes de pasar al siguiente slide

Ejercicio 3 — Recursión
Solución

Suma recursiva de una lista

def suma_lista(lista):
    if len(lista) == 0:   # caso base
        return 0
    return lista[0] + suma_lista(lista[1:])

print(suma_lista([1, 2, 3, 4]))  # 10
print(suma_lista([]))             # 0

# ¿Cómo funciona?
# suma([1,2,3,4])
# → 1 + suma([2,3,4])
# → 1 + 2 + suma([3,4])
# → 1 + 2 + 3 + suma([4])
# → 1 + 2 + 3 + 4 + suma([])
# → 1 + 2 + 3 + 4 + 0 = 10

Puntos clave

  • lista[1:] crea una sublista sin el primer elemento
  • Cada llamada el problema es 1 elemento más pequeño
  • La pila se va resolviendo de adentro hacia afuera
